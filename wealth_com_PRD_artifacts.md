## 9. Replication Cheat Sheet (For Building a Wealth.com Clone)

### Core entity model (inferred from screenshots)
1. **Client/Household** (top-level)
2. **People & Entities** — natural persons + LLCs/Trusts/Foundations
3. **Holdings & Assets** — typed (cash, equities, real estate, crypto, business interests)
4. **Financial Accounts** — linked via integrations
5. **Insurance & Liabilities**
6. **Documents & Records** — uploaded PDFs (Wills, Trusts, POAs)
7. **Tax Profile & Forms** — 1040, W-2, state forms, prior-year safe harbor
8. **Plans & Recommendations** — scenarios, projections
9. **Activity & Audit** — change log
10. **Tasks** — assignable to specific people (Jonathan, Kimberly visible in UI)

### Core workflows
1. **Estate Drafting** (Document Creation): pick documents → assign principals/agents → educate client → generate
2. **Estate Scan/Extract** (Ester AI): Upload PDF → Extract → Review → Save Contacts → Create Trust Card → Finish
3. **Tax Baseline**: Voice/manual/upload → generate baseline → Look Back / Look Forward
4. **Scenario Builder**: Pick assumptions → choose scenarios (GRAT/SLAT/ILIT/CLAT/Dynasty) → compare side-by-side
5. **Reports**: Generate Executive Summary, Standalone Visual Report

### Key UX patterns observed
- Sidebar nav with collapsible sections
- "Insight" pills surfaced as overlays on dashboards (e.g., "Washington State Law Changed")
- Workflow stepper bars (Upload → Extract → Review → Save → Create → Finish)
- Three-tab document viewer (PDF Preview / Document Highlights / Extraction Results)
- Comparison bar charts for scenario outcomes
- Action chips: "Assign as Task", "Add to Tracker"
- Persona-tagged feature pages (Enterprises = orange, Trust Companies = yellow, Attorneys = pink)
